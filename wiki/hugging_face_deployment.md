# Hugging Face Spaces Deployment Guide

This guide covers deploying the GEO Multi-omics Search app to Hugging Face Spaces. The app accepts plain-English dataset queries, greps the search index shards on disk, and passes filtered candidates to Claude Haiku for ranking and interpretation.

## How it works

1. User types a natural-language query (e.g. *"single-cell RNA-seq from APP/PS1 mice, matched in size to a 5XFAD dataset"*)
2. The app detects which shard(s) to search based on modality signals in the query
3. Python filters the shard line-by-line, scoring records by how many query terms they match
4. The top ~15–300 matching records are passed to Claude Haiku
5. Haiku returns a formatted table with GEO links and a brief explanation

The full 52 MB bulk RNA-seq shard never loads into LLM context — only grep results do. Each query sends ~500–5,000 tokens to Haiku, costing roughly $0.0002–$0.001.

## Files

The app lives in `spaces/` in this repo:

| File | Purpose |
|---|---|
| `spaces/app.py` | Main Gradio app — search logic + UI |
| `spaces/requirements.txt` | Python dependencies (`anthropic`, `gradio`) |
| `spaces/README.md` | HF Space card (YAML frontmatter + description) |

## One-time setup

### 1. Create the HF Space

1. Go to [huggingface.co/new-space](https://huggingface.co/new-space)
2. Set **Space name** (e.g. `geo-multiomics-search`)
3. Select **Gradio** as the SDK
4. Set visibility to **Public**
5. Click **Create Space** — this initializes a git repo at `https://huggingface.co/spaces/<username>/geo-multiomics-search`

### 2. Clone the Space repo

```bash
git clone https://huggingface.co/spaces/<your-username>/geo-multiomics-search
cd geo-multiomics-search
```

### 3. Copy app files from this repo

```bash
# From the GEO_llm repo root:
cp spaces/app.py /path/to/geo-multiomics-search/
cp spaces/requirements.txt /path/to/geo-multiomics-search/
cp spaces/README.md /path/to/geo-multiomics-search/
```

### 4. Copy the search index shards

The app expects the shards at `wiki/` relative to `app.py`. Create the folder and copy:

```bash
mkdir -p /path/to/geo-multiomics-search/wiki

# Standard assay shards
cp wiki/search_index_chipseq.txt         /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_atacseq.txt         /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_cut_run_tag.txt     /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_methylation.txt     /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_multiomics.txt      /path/to/geo-multiomics-search/wiki/

# RNA-seq modality splits (sized for fast load)
cp wiki/search_index_rnaseq_singlecell.txt  /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_rnaseq_snrnaseq.txt    /path/to/geo-multiomics-search/wiki/
cp wiki/search_index_rnaseq_spatial.txt     /path/to/geo-multiomics-search/wiki/

# Bulk RNA-seq — large file, needs Git LFS (see note below)
cp wiki/search_index_rnaseq.txt          /path/to/geo-multiomics-search/wiki/
```

**Note on the bulk RNA-seq shard:** `search_index_rnaseq.txt` is 52.9 MB. HF Spaces supports files up to 10 GB via Git LFS. Track it before committing:

```bash
cd /path/to/geo-multiomics-search
git lfs install
git lfs track "wiki/search_index_rnaseq.txt"
git add .gitattributes
```

Also track the ChIP-seq shard (13.9 MB) if you want to stay well under GitHub's regular file limit:

```bash
git lfs track "wiki/search_index_chipseq.txt"
```

### 5. Commit and push

```bash
cd /path/to/geo-multiomics-search
git add .
git commit -m "Initial deployment"
git push
```

HF Spaces will detect the `app.py` + `requirements.txt` and build automatically. The build takes 1–2 minutes.

### 6. Set the API key secret

1. Go to your Space on huggingface.co → **Settings** → **Repository secrets**
2. Add a secret named `ANTHROPIC_API_KEY` with your Anthropic API key value
3. The Space will restart and pick up the key — no rebuild needed

Your app will be live at:
```
https://huggingface.co/spaces/<your-username>/geo-multiomics-search
```

## Updating after a pipeline rebuild

When new GEO data is ingested and the search index is regenerated, update the Space with the new shards:

```bash
cd /path/to/geo-multiomics-search

# Copy updated shards from GEO_llm repo
cp /path/to/GEO_llm/wiki/search_index_*.txt wiki/

git add wiki/
git commit -m "Update search index shards — <date>"
git push
```

The modality-split RNA-seq files (`search_index_rnaseq_singlecell.txt` etc.) are auto-regenerated by `scripts/build_search_index.py` — no separate step needed.

## Keeping the Space in sync automatically (optional)

HF Spaces supports a GitHub Actions integration that can push to the Space on every commit to `main`. Add this workflow to `.github/workflows/sync-hf-space.yml` in the GEO_llm repo:

```yaml
name: Sync to HF Space
on:
  push:
    branches: [main]
    paths:
      - "spaces/**"
      - "wiki/search_index_*.txt"

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          lfs: true

      - name: Push to HF Space
        env:
          HF_TOKEN: ${{ secrets.HF_TOKEN }}
        run: |
          git config user.email "ci@github.com"
          git config user.name "GitHub Actions"

          # Clone Space repo
          git clone https://user:$HF_TOKEN@huggingface.co/spaces/<your-username>/geo-multiomics-search hf_space
          cd hf_space

          # Copy app files
          cp ../spaces/app.py .
          cp ../spaces/requirements.txt .
          cp ../spaces/README.md .

          # Copy shards
          mkdir -p wiki
          cp ../wiki/search_index_*.txt wiki/

          git add .
          git diff --cached --quiet || git commit -m "Sync from GitHub — $(date -u +%Y-%m-%d)"
          git push
```

Add `HF_TOKEN` (a HF write token) as a GitHub Actions secret in the GEO_llm repo settings.

## Cost estimate

Claude Haiku pricing (as of early 2026): $0.80 / 1M input tokens, $4.00 / 1M output tokens.

| Query type | Candidates sent | Approx input tokens | Cost per query |
|---|---|---|---|
| Specific (APP/PS1 scRNA-seq) | ~15 lines | ~2,000 | ~$0.0016 |
| Moderate (mouse kidney bulk) | ~100 lines | ~10,000 | ~$0.008 |
| Broad (no filters) | ~300 lines | ~30,000 | ~$0.024 |

At 100 queries/day of mixed specificity, expect **< $1/day**. Broader queries cost more — the intersection scoring in `app.py` is tuned to keep most queries in the specific–moderate range.

## Troubleshooting

**Space builds but shows "Error" on startup**
- Check the Space logs (Logs tab) — usually a missing dependency or import error
- Verify `requirements.txt` lists `anthropic>=0.40.0` and `gradio>=5.0.0`

**"ANTHROPIC_API_KEY not found" error**
- Set the secret in Space Settings → Repository secrets (not as a regular variable)
- Restart the Space after adding the secret

**Search returns no results**
- The shard file may not have been copied to `wiki/` in the Space repo
- Check that `wiki/search_index_rnaseq_singlecell.txt` etc. exist in the Space

**Bulk RNA-seq queries return nothing**
- Confirm `wiki/search_index_rnaseq.txt` was committed with Git LFS
- Run `git lfs ls-files` in the Space repo to verify

**LFS push rejected**
- HF Spaces free tier supports LFS — ensure you ran `git lfs install` and `git lfs track` before adding the file
